## نظرة عامة

فحص شامل للنظام مع إصلاح الأخطاء المتراكمة، ثم إضافة ميزتين مطلوبتين:
1. **ملف المحل** (Store Profile) محفوظ في قاعدة البيانات بدل localStorage
2. **طرق الدفع** (نقدي/بنكي) على الفواتير مع تفاصيل الحساب البنكي

---

## 1) فحص النظام وإصلاح الأخطاء

### أ) فحص أمني وقاعدة بيانات
- تشغيل `supabase--linter` لرصد مشاكل RLS والصلاحيات والفهارس
- التحقق من صحة الـ GRANT على كل جدول
- مراجعة سياسات RLS للتأكد من عدم تسرّب بيانات بين المستخدمين

### ب) فحص الواجهة والتنقل
- التأكد من عمل جميع الروابط في `AppSidebar` بعد إعادة تصميم الواجهة
- التحقق من عدم وجود صفحات فارغة أو placeholder
- فحص التوافق مع RTL في كل الصفحات

### ج) فحص التدفقات الحساسة
- تسجيل الدخول والخروج
- إنشاء فاتورة → خصم المخزون → إشعار انخفاض المخزون
- المرتجعات → إعادة المخزون
- صلاحيات المدير (بعد إصلاح "Auth session missing")

---

## 2) ملف المحل في قاعدة البيانات

**جدول جديد:** `store_profile` (سجل واحد لكل مستخدم)
- اسم المحل، الهاتف، العنوان، الرقم الضريبي، العملة
- شعار المحل (اختياري)
- إعدادات الفاتورة: رأس، تذييل، إظهار الشعار/الضريبة/QR
- إعدادات الطباعة: الحجم، عدد النسخ، طباعة تلقائية

**RLS:** كل مستخدم يرى ويعدّل ملفه فقط.

**واجهة:** تحديث `src/routes/settings.tsx` ليقرأ ويكتب من قاعدة البيانات بدل localStorage، مع ترحيل تلقائي للبيانات القديمة الموجودة محلياً.

---

## 3) طرق الدفع (نقدي / بنكي)

### أ) جدول جديد: `payment_methods`
لتخزين حسابات الدفع البنكية للمحل:
- الاسم (مثلاً: بنك الخرطوم — حساب رئيسي)
- النوع: `cash` أو `bank`
- اسم البنك، رقم الحساب، اسم صاحب الحساب
- IBAN (اختياري)
- نشط/غير نشط
- افتراضي (طريقة الدفع الأساسية)

**RLS:** لكل مستخدم طرق دفعه الخاصة.

### ب) تحديث جدول `invoices`
إضافة عمودين:
- `payment_method` نص: `cash` | `bank` | `mixed` (افتراضي: `cash`)
- `payment_method_id` UUID nullable → مرجع اختياري لطريقة الدفع المستخدمة

### ج) واجهة إدارة طرق الدفع
صفحة جديدة `/payment-methods` تحت الإعدادات:
- قائمة بطرق الدفع المسجّلة
- إضافة/تعديل/حذف
- تعيين طريقة افتراضية

### د) تحديث الكاشير والفواتير
- اختيار طريقة الدفع عند إنشاء الفاتورة (نقدي افتراضي / بنكي مع اختيار الحساب)
- عرض طريقة الدفع في تفاصيل الفاتورة `/invoices/:id`
- إظهار تفاصيل الحساب البنكي على الفاتورة المطبوعة عند اختيار "بنكي"

## تفاصيل تقنية

**Migrations:**
```
- create table public.store_profile (user_id unique, ...) + GRANTs + RLS
- create table public.payment_methods (...) + GRANTs + RLS  
- alter table public.invoices add column payment_method, payment_method_id
- triggers update_updated_at_column على الجدولين
```

**ملفات جديدة:**
- `src/routes/payment-methods.tsx`
- `src/hooks/use-store-profile.ts`
- `src/hooks/use-payment-methods.ts`

**ملفات معدّلة:**
- `src/routes/settings.tsx` — قراءة/كتابة من DB
- `src/routes/cashier.tsx` — اختيار طريقة الدفع
- `src/routes/invoices.$invoiceId.tsx` — عرض طريقة الدفع + تفاصيل بنكية
- `src/components/AppSidebar.tsx` — إضافة رابط طرق الدفع

**التبعيات:** لا حاجة لحزم جديدة.
