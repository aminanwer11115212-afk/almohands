## مبدأ التنفيذ

كل صفحة تصبح **وحدة مستقلة كاملة**:
- Hook خاص بها (`use-X.ts`) مسؤول عن CRUD والـ cache invalidation
- Route file يستخدم الـ hook فقط (لا استعلامات مباشرة)
- ربط تلقائي مع الصفحات المرتبطة (invalidateQueries)

## الدفعات

### 🟢 الدفعة 1 — إصلاحات سريعة عاجلة (هذه الجلسة)

**1.1 تقريب الأسعار لأعلى لأقرب 100** في صفحة `prices.tsx`
- الصيغة: `Math.ceil(newPrice / 100) * 100`
- مثال: `1250 → 1300`، `1201 → 1300`، `1000 → 1000`

**1.2 ربط صفحة `prices.tsx` بقاعدة البيانات**
- قراءة التصنيفات ديناميكياً من `products.category`
- تنفيذ فعلي للتحديث الجماعي (UPDATE على products)
- عرض عدد المنتجات المتأثرة + تأكيد قبل التنفيذ
- عرض معاينة (كم منتج، متوسط السعر قبل/بعد)

**1.3 إعادة تصميم شاشة الكاشير (POS)** — نمط عملي احترافي:
```text
┌─────────────────────────────────┬──────────────────┐
│  🔍 بحث سريع (فوكس تلقائي)      │   السلة (سطرية)  │
│  ┌─── شبكة منتجات كبيرة ───┐    │   منتج × كم = س  │
│  │ [منتج] [منتج] [منتج]    │    │   ────────────   │
│  │ [منتج] [منتج] [منتج]    │    │   الإجمالي: ٥٠٠  │
│  └───────────────────────────┘  │   [كاش][بنك]    │
│  فلاتر تصنيف أفقية              │   [💰 دفع F2]   │
└─────────────────────────────────┴──────────────────┘
```
- اختصارات لوحة مفاتيح: F2 دفع، Esc إلغاء، Enter بحث
- إضافة سريعة بالنقر (كمية = 1 تلقائياً، +/- للتعديل)
- كيبورد رقمي للـ tablet
- عرض المخزون المتبقي على كل منتج
- دعم عميل نقدي/مسجّل بنقرة واحدة

### 🟡 الدفعة 2 — ربط الصفحات المعزولة (الجلسة التالية)

**2.1 صفحة العملاء (`customers.tsx`)**
- Hook: `use-customers.ts` (موجود بالفعل — نتحقق ونستخدمه)
- قائمة + بحث + إضافة/تعديل/حذف
- عرض إجمالي المديونية لكل عميل
- زر "عرض الفواتير" يفتح `/invoices?customer_id=`

**2.2 صفحة المصروفات (`expenses.tsx`)**
- Hook: `use-expenses.ts` (موجود)
- قائمة + فلترة بالتاريخ + إضافة (نوع، مبلغ، ملاحظة)
- ربط مع طريقة الدفع (كاش/بنك)
- ملخص الشهر الحالي

**2.3 صفحة المرتجعات (`returns.tsx`)**
- Hook: `use-returns.ts` (موجود)
- ربط مباشر بالفواتير: اختر فاتورة → اختر منتجات → إرجاع
- Trigger DB الموجود `restore_stock_on_return_accepted` يعيد المخزون تلقائياً
- عرض حالة (pending/accepted/rejected)

**2.4 صفحة الموردين (`suppliers.tsx`)**
- تحقق من الربط وإكماله

## تفاصيل تقنية

**تقريب الأسعار:**
```typescript
function roundUpToHundred(n: number): number {
  return Math.ceil(n / 100) * 100;
}
// في التحديث الجماعي:
const factor = dir === "inc" ? 1 + percent/100 : 1 - percent/100;
const newPrice = roundUpToHundred(oldPrice * factor);
```

**التحديث الجماعي عبر Supabase:**
- استعلام المنتجات المطابقة للتصنيف
- تحديث كل منتج بالسعر الجديد المقرَّب
- استخدام `.update()` مع فلتر أو RPC function للأداء

**بنية الكاشير الجديدة:**
- CSS Grid: `grid-cols-[1fr_360px]` على desktop
- Sticky cart panel على اليمين
- Debounced search (150ms)
- Virtualized product grid لو المنتجات >100
